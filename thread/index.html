线程

  • 引入线程
    • 进程是资源分配的最小单位
    • 线程是调度的最小单位
    • 线程共享进程的所有资源
  • 为什么要引入线程
    • 切换进程CPU高速缓存cache内容丢失
    • 切换进程TLB失效，开销大
    • 进程之间通信，需要打破隔离的壁障
      • 线程之间通信，开销小
    • 进程的创建与销毁比较耗时
      • 线程的创建与销毁要轻量很多

获取线程的标识

PTHREAD_SELF(3)                  Linux Programmer's Manual                 PTHREAD_SELF(3)

NAME
       pthread_self - obtain ID of the calling thread

SYNOPSIS
       #include <pthread.h>

       pthread_t pthread_self(void);
------------------------------------------------------------------------------------------------------------
       Compile and link with -pthread.  //使用线程的时候，编译命令一定要加 -pthread 或者 -lpthread  选项
------------------------------------------------------------------------------------------------------------
RETURN VALUE
       This function always succeeds, returning the calling thread's ID.//函数永远成功，返回线程ID
------------------------------------------------------------------------------------------------------------

创建线程

PTHREAD_CREATE(3)                Linux Programmer's Manual               PTHREAD_CREATE(3)

NAME
       pthread_create - create a new thread

SYNOPSIS
       #include <pthread.h>
------------------------------------------------------------------------------------------------------------
       int pthread_create(pthread_t *thread, const pthread_attr_t *attr,
                          void *(*start_routine) (void *), void *arg);
//thread:创建子线程的线程ID
//attr:线程属性，一般填NULL，表示使用默认属性
//start_routine :线程的入口函数
//arg:入口函数的参数
------------------------------------------------------------------------------------------------------------
       Compile and link with -pthread.
------------------------------------------------------------------------------------------------------------
RETURN VALUE
       On success, pthread_create() returns 0; on error, it returns an error  number,  and
       the contents of *thread are undefined.
//成功返回0，错误返回错误码，不会设置errno

条件变量（pthread_cond_t）

条件变量只是提供了一个等待唤醒的机制

pthread_cond_init

PTHREAD_COND(3)                 Library Functions Manual                                    PTHREAD_COND(3)

NAME
       pthread_cond_init, pthread_cond_destroy, pthread_cond_signal, pthread_cond_broadcast, pthread_cond_wait, pthread_cond_timedwait - operations on conditions

SYNOPSIS
       #include <pthread.h>

       pthread_cond_t cond = PTHREAD_COND_INITIALIZER;

       int pthread_cond_init(pthread_cond_t *cond, pthread_condattr_t *cond_attr);
------------------------------------------------------------------------------------------------------------
       int pthread_cond_signal(pthread_cond_t *cond);//唤醒一个满足该条件的线程（可能唤醒多个）
------------------------------------------------------------------------------------------------------------
       int pthread_cond_broadcast(pthread_cond_t *cond);//唤醒所有满足该条件的线程
------------------------------------------------------------------------------------------------------------
       int pthread_cond_wait(pthread_cond_t *cond, pthread_mutex_t *mutex);//无限期等待
------------------------------------------------------------------------------------------------------------
       int pthread_cond_timedwait(pthread_cond_t *cond, pthread_mutex_t *mutex, const struct timespec *abstime);//超时时间
------------------------------------------------------------------------------------------------------------
       int pthread_cond_destroy(pthread_cond_t *cond);

Question：pthread_cond_wait()为什么需要传递互斥锁？

Answei：保护条件变量或者其他共享资源

当条件不成立，一直等待

int pthread_cond_wait(pthread_cond_t *cond, pthread_mutex_t *mutex);

当条件不成立，不超过超时时间等待

int pthread_cond_timedwait(pthread_cond_t *cond, pthread_mutex_t *mutex, const struct timespec *abstime);

语义：当wait返回时，条件曾经成立（目前是否成立还需检测）

              ^
             | |
             | |-------多个线程共享

等待某个条件成立的操作步骤

释放mutex ------------------------- 
                                |                  
                             原子操作
                                |
陷入阻塞状态------------------------
当返回时，该线程一定再依次获取了mutex

当条件成立时，唤醒所有等待的线程

int pthread_cond_broadcast(pthread_cond_t *cond);

唤醒一个等待该条件的线程（在实际实现中，为了性能，可能唤醒多个线程）

int pthread_cond_signal(pthread_cond_t *cond);

生产者消费者模型

用户：生产请求

服务端：回应请求（消费请求）


    生产者-------->[       阻塞队列      ]------------->消费者、消费者2.......
    生产者2-----^    
    生产者3----^
      ......

阻塞队列：当队列满时，如果线程往队列中添加东西，线程陷入阻塞；当队列空时，线程往队列中取东西，			线程陷入阻塞。
        
“生产者” :生产“商品”，将商品添加阻塞队列
    		如果队列满了，生产者陷入阻塞，等待队列不满（Not full）
    		如果队列不满，将商品添加到阻塞队列，队列非空（Not empty）,唤醒消费者
“消费者”：消费“商品”
    		如果队列空了，消费者陷入阻塞，等待队列非空（not empty）
    		如果队列非空，从阻塞队列中获取商品，队列不满（not full），唤醒生产者

Answer：取决于CPU的核数以及任务的负载（任务分为I/O密集型任务和计算密集型任务）

​ 如果时计算密集型任务，一般为1：1,如果是I/O密集型任务，一般为1:N,N>=2)
